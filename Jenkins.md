1. Jenkins is an open-source CI/CD tool. It helps build out CI/CD pipelines. 
2. Why Jenkins:
   1. Open-source
   2. Extensive plugin ecosystem
   3. Flexible with projects of any size and complexity (Scalable)
   4. Compatible with any OS and infrastructure (on-premises or cloud).
   5. Strong community support
   6. Easy to install and configure
   7. Define CI/CD pipeline as code
   8. A larger set of tools for monitoring and reporting
   9. Highly integratable with deployment and development tools.
3. Challenges without Jenkins
   1. Manual build and deployment process.
   2. Lack of consistency
   3. Delayed feedback
   4. Limited scalability with manual pipelines
   5. Poor visibility and traceability
   6. Difficulty in collaborating
   7. Increased risk of human error
   8. Resource-intensive (both time and computing)
   9. Lack of automated testing integration
   10. Slow release cycles
4. Jenkins jobs -> It is a set of instructions that you want Jenkins to perform.
5. A webhook is a way for one application to automatically notify another application when something happens.
